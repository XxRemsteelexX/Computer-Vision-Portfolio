Answer:

I designed my CNN architecture for landmark classification through four key steps:¶

Hierarchical Feature Extraction:¶

Created a progressive convolutional backbone (64→128→256→512 filters) to capture features from basic edges to complex landmark patterns.

Training Stability:¶

Added batch normalization and dropout (0.4) to stabilize training and prevent overfitting on a dataset with variable lighting and perspectives.

Classification Pathway:¶

Implemented a three-layer classifier to transform spatial features into landmark categories, with sufficient depth to learn complex relationships.

Computational Efficiency:¶

Used max pooling and adaptive pooling to balance feature preservation with processing speed.

This architecture provides enough capacity to distinguish between similar landmarks while avoiding excessive parameters, as validated by the 50% accuracy achieved without transfer learning.

from src.predictor import Predictor
from src.helpers import compute_mean_and_std

# First let's get the class names from our data loaders
class_names = data_loaders["train"].dataset.classes

# Then let's move the model_transfer to the CPU
# (we don't need GPU for inference)
model_transfer = model_transfer.cpu()
# Let's make sure we use the right weights by loading the
# best weights we have found during training
# NOTE: remember to use map_location='cpu' so the weights
# are loaded on the CPU (and not the GPU)
model_transfer.load_state_dict(
    torch.load("checkpoints/model_transfer.pt", map_location="cpu")
)

# Let's wrap our model using the predictor class
mean, std = compute_mean_and_std()
predictor = Predictor(model_transfer, class_names, mean, std).cpu()

# Export using torch.jit.script
scripted_predictor = torch.jit.script(predictor)
scripted_predictor.save("checkpoints/transfer_exported.pt")
